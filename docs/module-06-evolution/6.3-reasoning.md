Context-aware ([6.2 Context-Aware](6.2-context-aware.md)) AI models are powerful, but they are "fast" thinkers. They start talking immediately, word by word, without "planning" their answer first. 

This leads us to **Level 3: Reasoning & Feedback Loops**.

## What is a Level 3 AI?

A Level 3 AI is a model that can perform [Chain-of-Thought (CoT)](../glossary.md#chain-of-thought) reasoning. Instead of just answering your question, the model is given space to "think" about its answer *before* providing the final output.

### The Reasoning Loop

1.  **Thinking Phase:** The AI writes a hidden "internal draft" of how it should solve your problem. 
    *   Example: "Okay, the user wants a math proof. First I need to define the variables, then I'll use the dot product formula, and finally I'll calculate the final similarity."
2.  **Self-Correction:** The model can actually "read" its own thought process and fix errors. 
    *   Example: "Wait, I used the wrong formula for cosine similarity. Let me fix that before showing the final answer."
3.  **The Output:** Only after this reasoning is done does the AI provide you with the finished response.

## Why this is Revolutionary

By adding this extra "thinking step," Level 3 models can solve much more complex problems than Level 2 models. This is what you see in specialized reasoning models like OpenAI's **o1** or DeepSeek **R1**. 

These models can:
*   Write complex software code with fewer bugs.
*   Solve multi-step math and logic puzzles.
*   "Self-evaluate" their work to ensure it's accurate.

## Visualizing Level 3 in Python

```python
def level_3_reasoning(query):
    # Phase 1: The Hidden Thought Process
    print("[Internal Thought]: The user wants to calculate a 2D dot product.")
    print("[Internal Thought]: Formula is (a1 * b1) + (a2 * b2).")
    
    # Phase 2: A deliberate calculation step
    a = [1, 2]
    b = [3, 4]
    result = (a[0] * b[0]) + (a[1] * b[1])
    
    print("[Internal Thought]: Result is 11. Wait, let me check... 1*3=3, 2*4=8, 3+8=11. Looks correct.")

    # Phase 3: The final answer to the user
    return f"The dot product of [1, 2] and [3, 4] is {result}."

print(level_3_reasoning("Calculate the dot product."))
```

Running this prints:

```text
[Internal Thought]: The user wants to calculate a 2D dot product.
[Internal Thought]: Formula is (a1 * b1) + (a2 * b2).
[Internal Thought]: Result is 11. Wait, let me check... 1*3=3, 2*4=8, 3+8=11. Looks correct.
The dot product of [1, 2] and [3, 4] is 11.
```

## The Limitations of Level 3

While Level 3 AI can "think," it still only produces *text*. It doesn't actually **do** anything. It can think about how to write a file, but it can't actually create that file on your computer. 

To bridge that gap, we need to empower the AI with actions. This brings us to Level 4.

**Exercise 2:** Why does [Chain-of-Thought](../glossary.md#chain-of-thought) reasoning work better for complex problems than direct answers?

**Answer:** Chain-of-Thought forces the model to break problems into smaller steps. By showing reasoning step-by-step, the model can catch its own mistakes (self-correction) and doesn't have to predict the final answer in one shot. This reduces error propagation—an error in step 1 can be caught before step 2. For complex math or logic problems, this explicit intermediate reasoning is often crucial for correctness.
