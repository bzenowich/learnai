Our mini chatbot will:

2. **Retrieve** the most relevant documents for a question using dot-product

3. **Assemble** a prompt from a system instruction, the retrieved context, and the
   user's question.
4. **Generate** an answer (first with a simple stand-in, then — optionally — with a

5. Wrap it all in a small **agent-style loop**.

```python
]
```


## Step 2: A toy embedding function

A real [embedding](../glossary.md#embedding) model is a giant neural network. To
keep everything runnable offline, we'll build a **bag-of-words** embedding: we fix
a small vocabulary, then represent each document as a vector counting how often
each vocabulary word appears. It's crude, but it captures the key idea — *text
becomes a vector you can compare with a dot product*.

We also drop very common filler words ("the", "of", "how"…) — called **stopwords**
— so only meaningful words shape the vector. Otherwise every sentence would look
similar just because they all contain "the".

```python
import numpy as np
import re

# Common words that carry little meaning; we ignore them when embedding.
STOPWORDS = {
    "the", "a", "an", "of", "to", "and", "is", "are", "in", "on", "how",
    "does", "do", "what", "it", "that", "this", "with", "for", "as", "into",
}

def tokenize(text):
    # Lowercase, split into words, and drop stopwords (see Module 2.1 Tokenization).
    return [w for w in re.findall(r"[a-z]+", text.lower()) if w not in STOPWORDS]

# Build a vocabulary from every meaningful word in the knowledge base.
vocabulary = sorted({word for doc in documents for word in tokenize(doc)})
word_to_index = {word: i for i, word in enumerate(vocabulary)}

def embed(text):
    """Turn text into a word-count vector over the fixed vocabulary."""
    vector = np.zeros(len(vocabulary))
    for word in tokenize(text):
        if word in word_to_index:
            vector[word_to_index[word]] += 1.0
    return vector

print(f"Vocabulary size: {len(vocabulary)}")
print(f"Words matched in 'dot product of vectors': {int(embed('dot product of vectors').sum())}")
```

Running this prints:

```text
Vocabulary size: 40
Words matched in 'dot product of vectors': 3
```

## Step 3: Retrieve with cosine similarity

To find the most relevant documents we compare the question's embedding to each
document's embedding. We use [cosine similarity](../glossary.md#cosine-similarity)
— a [dot product](../glossary.md#dot-product) normalized by length — exactly as
described in [2.3 Vector Space & Similarity](../module-02-text/2.3-vector-space-similarity.md).

```python
# Pre-compute an embedding for every document once.
document_embeddings = np.array([embed(doc) for doc in documents])

def cosine_similarity(a, b):
    denom = np.linalg.norm(a) * np.linalg.norm(b)
    return 0.0 if denom == 0 else float(np.dot(a, b) / denom)

def retrieve(question, top_k=2):
    """Return the top_k most similar documents to the question."""
    query_vector = embed(question)
    scores = [cosine_similarity(query_vector, doc_vec) for doc_vec in document_embeddings]
    ranked = sorted(range(len(documents)), key=lambda i: scores[i], reverse=True)
    return [(documents[i], scores[i]) for i in ranked[:top_k]]

question = "How does an attention mechanism use scores?"
for doc, score in retrieve(question):
    print(f"{score:.3f}  {doc}")
```

Running this prints:

```text
0.267  Attention lets a transformer weigh the importance of each token.
0.267  Softmax turns a list of scores into probabilities that sum to one.
```

The retriever found the two most relevant facts using nothing but dot products —
this is the heart of [RAG](../glossary.md#retrieval-augmented-generation).

## Step 4: Assemble the prompt

Now we build the [prompt](../glossary.md#prompt) the way
[4.1 Anatomy of a Prompt](../module-04-context/4.1-anatomy-of-a-prompt.md)
describes: a [system prompt](../glossary.md#system-prompt) that sets the rules,
the retrieved context, and the user's question.

```python
SYSTEM_PROMPT = (
    "You are a helpful tutor. Answer using ONLY the provided context. "
    "If the context doesn't contain the answer, say you don't know."
)

def build_prompt(question, retrieved):
    context = "\n".join(f"- {doc}" for doc, _ in retrieved)
    return (
        f"[SYSTEM]\n{SYSTEM_PROMPT}\n\n"
        f"[CONTEXT]\n{context}\n\n"
        f"[QUESTION]\n{question}"
    )

print(build_prompt(question, retrieve(question)))
```

Running this prints:

```text
[SYSTEM]
You are a helpful tutor. Answer using ONLY the provided context. If the context doesn't contain the answer, say you don't know.

[CONTEXT]
- Attention lets a transformer weigh the importance of each token.
- Softmax turns a list of scores into probabilities that sum to one.

[QUESTION]
How does an attention mechanism use scores?
```

## Step 5: Generate an answer (offline stand-in)

A real chatbot would send that prompt to an LLM. So we can run everything offline,
we'll start with a deliberately simple "generator": it just returns the single
most relevant document. This is *not* how a real model works — it's a placeholder
so we can test the whole pipeline before adding a real model in Step 6.

```python
def generate_offline(prompt, retrieved):
    """A stand-in for an LLM: return the top retrieved fact as the 'answer'."""
    if not retrieved or retrieved[0][1] == 0.0:
        return "I don't know based on the provided context."
    return f"Based on the context: {retrieved[0][0]}"

retrieved = retrieve(question)
print(generate_offline(build_prompt(question, retrieved), retrieved))
```

Running this prints:

```text
Based on the context: Attention lets a transformer weigh the importance of each token.
```

## Step 6: Wrap it in an agent-style loop

Finally, we tie retrieval, prompting, and generation together into one function —
the same shape as the [agent](../glossary.md#agent) loop from
[6.4 Agentic AI](../module-06-evolution/6.4-agentic-ai.md): take input, gather
context (a "tool" call), produce a response.

```python
def chatbot(question, generate=generate_offline):
    retrieved = retrieve(question)
    prompt = build_prompt(question, retrieved)
    return generate(prompt, retrieved)

for q in [
    "What does softmax do?",
    "How do agents take actions?",
    "What is the capital of France?",
]:
    print(f"Q: {q}")
    print(f"A: {chatbot(q)}\n")
```

Running this prints:

```text
Q: What does softmax do?
A: Based on the context: Softmax turns a list of scores into probabilities that sum to one.

Q: How do agents take actions?
A: Based on the context: An agent uses tools in a loop to take actions in the world.

Q: What is the capital of France?
A: I don't know based on the provided context.
```

Notice the last answer: because no document mentions France, the retriever finds
nothing relevant and the bot honestly says it doesn't know — exactly the behavior
RAG is designed to produce, and the antidote to
[hallucination](../glossary.md#hallucination) from
[5.1 The Limits of Memory](../module-05-rag-tools/5.1-limits-of-memory.md).

## Step 7 (optional): Use a real Claude model

The only toy part of our chatbot is the generator. Swapping in a real
[LLM](../glossary.md#transformer) is a small change: send the same assembled
prompt to a model and return its reply. The function below is a drop-in
replacement for `generate_offline`.

To run it you'll need an Anthropic API key:

```bash
pip install anthropic
export ANTHROPIC_API_KEY="your-key-here"
```

```python
def generate_with_claude(prompt, retrieved, model="claude-opus-4-8"):
    """Drop-in replacement for generate_offline that calls a real Claude model.

    Requires `pip install anthropic` and the ANTHROPIC_API_KEY environment
    variable. The import is lazy so this lesson still runs without the package.
    """
    import anthropic  # imported here so the rest of the lesson runs offline

    context = "\n".join(f"- {doc}" for doc, _ in retrieved)
    client = anthropic.Anthropic()  # reads ANTHROPIC_API_KEY from the environment
    response = client.messages.create(
        model=model,                # the latest, most capable Claude model
        max_tokens=1024,
        system=SYSTEM_PROMPT,
        messages=[{"role": "user", "content": f"Context:\n{context}\n\nQuestion: {prompt}"}],
    )
    return "".join(block.text for block in response.content if block.type == "text")

# To use it, just pass it in:  chatbot("What does softmax do?", generate=generate_with_claude)
print("generate_with_claude() is ready — set ANTHROPIC_API_KEY, then pass it to chatbot().")
```

Running this prints:

```text
generate_with_claude() is ready — set ANTHROPIC_API_KEY, then pass it to chatbot().
```

With a real model, the chatbot will write fluent answers grounded in the retrieved
context instead of echoing a single fact — and it will still say "I don't know"
when the context can't support an answer, because the system prompt told it to.
(For a quicker, cheaper option you can swap `model="claude-haiku-4-5"`.)
